## 1. 引言

### 1.1 目的

本文档旨在详细描述AI测试用例生成系统（AiToCase V2.0）的功能需求、非功能需求、用户界面需求以及数据处理需求，作为开发、测试和验收的基准文档。

### 1.2 范围

AiToCase V2.0 是一款基于FastAPI框架和多种大语言模型的智能测试用例自动生成平台，支持多种需求输入方式和多格式导出功能。

### 1.3 定义、缩略语

| 术语 | 定义 |
|------|------|
| LLM | 大语言模型 (Large Language Model) |
| SSE | 服务器发送事件 (Server-Sent Events)，用于流式输出 |
| OCR | 光学字符识别 (Optical Character Recognition) |
| 向量数据库 | 存储文本向量表示的数据库，用于语义检索 |
| Axure | Axure RP，一款原型设计工具 |
| Chroma | 开源向量数据库 |
| LlamaIndex | 开源索引框架 |

### 1.4 参考资料

- FastAPI官方文档
- LlamaIndex官方文档
- Chroma官方文档
- 各LLM提供商API文档（DeepSeek、阿里云、字节跳动、微软Azure）

## 2. 系统概述

### 2.1 系统背景

随着软件项目规模的扩大，传统的手工编写测试用例方式效率低下，难以满足快速迭代的需求。本系统旨在利用人工智能技术，从多种形式的需求文档（PDF、Word、Markdown、图片、Axure原型等）自动生成结构化测试用例，大幅提升测试用例编写效率。

### 2.2 系统目标

1. 支持多种格式的需求文档解析
2. 实现基于AI的智能测试用例生成
3. 提供向量检索功能，支持需求语义搜索
4. 支持多种格式的测试用例导出
5. 提供知识库管理功能，支持增量需求管理

### 2.3 系统用户

| 用户角色 | 描述 |
|----------|------|
| 测试工程师 | 使用系统生成测试用例 |
| 产品经理 | 上传需求文档，验证生成结果 |
| 开发人员 | 查阅导出的测试用例 |

### 3.2 功能详细描述

#### 3.2.1 文档解析功能

**输入**：
- PDF文件（.pdf）
- Word文档（.docx, .doc）
- Markdown文件（.md）
- 文本文件（.txt）
- Axure文件（.zip, .html, .htm）
- 图片文件（.png, .jpg, .jpeg）
- Axure在线链接

**处理流程**：
1. 读取上传文件内容
2. 根据文件类型调用对应解析器
3. 提取文本内容
4. 如需OCR，调用视觉模型识别
5. 返回解析结果

**输出**：
- 纯文本内容
- 增量内容（针对Axure）
- 结构化Markdown

#### 3.2.2 向量检索功能

**输入**：
- 查询文本
- 集合名称
- top_k参数

**处理流程**：
1. 将查询文本转为向量
2. 在向量数据库中检索相似内容
3. 返回top_k个最相似的文档块

**输出**：
- 检索到的文本内容
- 相关性分数

#### 3.2.3 测试用例生成功能

**输入**：
- 需求描述文本
- 模块名称
- 测试用例数量
- LLM提供商选择

**处理流程**：
1. 构建系统提示词
2. 调用LLM API生成测试用例
3. 流式返回生成过程
4. 解析JSON返回结果

**输出**：
```json
{
  "test_cases": [
    {
      "test_module": "模块名称",
      "case_id": "TC-001",
      "priority": "高",
      "title": "测试用例标题",
      "precondition": "前置条件描述",
      "steps": "1. 步骤一\n2. 步骤二\n3. 步骤三",
      "expected_result": "1. 预期结果一\n2. 预期结果二"
    }
  ]
}
```

#### 3.2.4 知识库管理功能

**输入**：
- 知识库名称
- Axure URL
- 可选：用户名、密码

**处理流程**：
1. 通过无头浏览器获取Axure内容
2. 解析所有页面
3. 分离全量和增量内容
4. 向量化存储到Chroma
5. 建立页面关联关系

**输出**：
- 知识库集合
- 页面索引

## 4. 外部接口需求

| 接口名称 | 方法 | 描述 |

| 指标 | 要求 |
|------|------|

| 测试用例生成 | 流式响应，首字节≤3秒 |

| 指标 | 要求 |

| 指标 | 要求 |

## 8. 附录

### 8.1 技术栈

- **Web框架**: FastAPI + Uvicorn
- **模板引擎**: Jinja2
- **向量数据库**: LlamaIndex + Chroma
- **Embedding**: HuggingFace / 通义千问
- **LLM**: DeepSeek / 通义千问 / Azure OpenAI / 豆包
- **文档解析**: PyMuPDF + python-docx + BeautifulSoup
- **数据处理**: Pandas + NumPy

### 8.2 项目结构

```

├── main.py                 # FastAPI主程序入口
```
