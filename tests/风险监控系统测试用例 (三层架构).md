## 🎯 测试分层策略

```
Test Hierarchy:
├── Unit Tests (单元层)
│   ├── RiskRuleEngine
│   │   ├── SN重复检测 (单平台/跨平台)
│   │   ├── 地址超量检测
│   │   └── 地址批量雷同检测
│   ├── OrderValidator
│   │   ├── SN码格式校验
│   │   └── 地址区划校验
│   └── AuditLogger
│       ├── 日志记录
│       └── 防篡改验证
├── Integration Tests (整合层)
│   ├── RuleConfigAPI
│   │   ├── 配置保存/查询
│   │   └── 配置热更新
│   ├── OrderListAPI
│   │   ├── 列表查询/筛选
│   │   ├── 分页/导出
│   │   └── 批量处置
│   ├── OrderDetailAPI
│   │   ├── 详情查询
│   │   └── 处置操作
│   └── ExternalAPI
│       ├── 订单接收
│       └── 指令推送
└── E2E Tests (端到端层)
    ├── 正常业务流程
    ├── 风险拦截流程
    ├── 人工复核流程
    ├── 事后回溯流程
    ├── 权限控制流程
    └── 性能/稳定性
```
